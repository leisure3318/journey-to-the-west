# 宝箱系统设计

## 概述

地图上随机生成宝箱，玩家靠近后自动开启，获得法器奖励。鼓励探索地图，为build搭配增加随机性。

## 宝箱生成规则

| 参数 | 值 |
|------|------|
| 总数量 | 8-12个/局 |
| 分布 | 均匀散布地图，避开 POI 半径 200 以内 |
| 触发半径 | 40px（靠近自动开启） |
| 视觉 | 程序化绘制的小宝箱（棕色底+金色锁），迷雾覆盖时不可见 |
| 开箱动画 | 宝箱打开 → 法器图标弹出放大 → 名称+效果文字浮现 → 淡出 |

## 宝箱视觉

程序化生成纹理（BootScene）：
- 棕色木箱底座（矩形）
- 金色锁扣（小矩形）
- 开启后：盖子翻开动画 → 光柱 → 物品弹出

## 实现要点

1. **ChestSystem.ts**：管理宝箱生成、碰撞检测、开箱逻辑
2. 宝箱位置在 GameScene.create 时随机生成，避开已有 POI
3. 每个宝箱只能开一次，开后消失
4. 法器效果通过修改 UpgradeState 实现（复用现有属性字段）
5. 需要的法器如果关联未招募的徒弟（如悟空不在时开到如意金箍），重新随机
6. 宝箱在小地图上显示为黄色小点（开启前）
