昨天我从一个"西游记×吸血鬼幸存者"的想法出发，用AI生成了3份设计文档和119张游戏素材。角色能在屏幕上跑了，但还不算游戏。

今天的目标：**让它真的能玩。**

## 技术栈：Phaser 4 + TypeScript

选 Phaser 4 做 Web 游戏引擎，理由很简单——轻量、2D友好、做完直接网页打开就能玩。打包工具用 Parcel 2，TypeScript 做类型检查。

Claude 直接帮我搭好了模块化的项目骨架。我跟它反复强调一个原则：**不要屎山，每个文件不超过200行。**

最终的项目结构长这样：

```
src/
├── config/
│   ├── GameConfig.ts          # 全局数值（HP、攻击力、冷却时间）
│   └── MapConfig.ts           # 地图POI配置（五指山、高老庄...）
├── entities/
│   ├── Player.ts              # 唐僧（移动、受伤、回血）
│   ├── Disciple.ts            # 徒弟（3种攻击方式）
│   ├── Enemy.ts               # 小怪（追踪、远程、自爆）
│   └── Boss.ts                # Boss（4阶段AI）
├── systems/
│   ├── CombatSystem.ts        # 战斗（投射物、伤害飘字、血条）
│   ├── EnemySpawner.ts        # 刷怪（4阶段递增）
│   ├── WukongAI.ts            # 悟空AI（出击/战斗/回归/护驾）
│   ├── DiscipleManager.ts     # 徒弟管理
│   ├── BossSystem.ts          # Boss战（地点触发、多Boss）
│   ├── RecruitmentSystem.ts   # 收徒系统（地标触发）
│   ├── LandmarkSystem.ts      # 地标绘制
│   ├── FogOfWar.ts            # 迷雾系统
│   └── DecorationSystem.ts    # 地图装饰
├── ui/
│   ├── HUD.ts                 # 血条/经验条/计时器
│   ├── LevelUpPanel.ts        # 升级3选1
│   ├── MiniMap.ts             # 小地图
│   ├── BossHpBar.ts           # Boss血条
│   └── ...                    # 暂停、结束、胜利面板
└── scenes/
    ├── BootScene.ts            # 资源加载
    ├── MenuScene.ts            # 主菜单（山水背景+行走动画）
    ├── CutsceneScene.ts        # 过场CG系统
    └── GameScene.ts            # 游戏主场景（纯协调层）
```

20个源文件，最大的也就200行。**模块化是跟AI协作的命脉**——你不设规矩，AI会把所有逻辑塞进一个文件里，三天后连它自己都看不懂。

## 核心玩法：从零到能玩

吸血鬼幸存者的核心循环：**移动 → 自动攻击 → 杀怪掉经验 → 升级选技能 → 怪越来越多**

### 唐僧移动 + 三徒弟攻击

WASD、方向键、鼠标点击，三种操控全支持。然后让三个徒弟各有各的打法：

- **悟空**：120°扇形近战，金色弧光横扫，自带AI状态机（发现妖怪冲上去打，打完自己飞回来）
- **八戒**：360°范围伤害+击退，遇到Boss自动冲上去当肉盾
- **沙僧**：远程投射物，自动瞄准最近敌人，可穿透3个目标

### 6种妖怪 + 精英怪

不同阶段解锁不同类型的妖怪：
- 山贼、猴妖（追着你跑）
- 弓箭手（保持距离射箭）
- 火精（冲到面前自爆）
- 蝎子精、熊妖（高阶段出现）

5%概率出精英怪——红色光环，3倍血量，5倍经验。看到红光先跑还是刚正面，就看你等级够不够了。

### 升级系统

杀怪掉绿色经验珠，升级弹出3选1：

**独行阶段**（没有徒弟时）：
- 佛光普照（回血）、健步如飞（移速）、金光护体（减伤）、般若心经（加血量上限）

**收徒后**：
- 金箍棒强化、棒扫千军、疾风棒（悟空三连强化）
- 金刚不坏、草上飞、金钵盂、锦斓袈裟、人参果

全是西游主题的命名，AI取名确实比我有文化。

## 重头戏：西游原著世界地图

最有意思的设计是**地图探索系统**——按原著设定，每个地点随机分布在3200×2400的大地图上：

**收徒地标：**
- 🏔 五指山 — 收孙悟空
- 🏘 高老庄 — 收猪八戒
- 🌊 流沙河 — 收沙悟净

**Boss地标：**
- ⚔ 黄风岭 — 黄风大王
- 💀 白虎岭 — 白骨精
- 🔥 火云洞 — 红孩儿

配合**迷雾系统**——未探索的区域是黑的，走过才会揭开。右上角有**小地图**，显示探索进度和已发现的地标位置（绿点=收徒，红点=Boss）。

**收徒是按顺序的**：必须先找到五指山收悟空，才能去高老庄收八戒。在独行阶段（没有徒弟时），不会触发Boss战——毕竟唐僧自己没有攻击手段，遇到Boss必死。

打败一个Boss不算结束，要**打完三个Boss才算取经成功**。

## 踩过的坑

### 坑1：Boss出场游戏卡死

这个bug最折腾。Boss出现的瞬间整个游戏冻住，反复出现了好几次。

**根因**：Phaser 4 会自动调用游戏对象的 `update()` 方法，但参数签名跟我自定义的不一样，导致物理计算出现NaN，整个引擎崩溃。

**解决**：
1. 把Boss的逻辑方法从 `update()` 改名为 `tick()`
2. Boss类从 `Phaser.Physics.Arcade.Sprite` 改为普通 `Phaser.GameObjects.Sprite`
3. 加了 try-catch 防护，即使出错也不会卡死

**教训**：不要在 Phaser 游戏对象上覆盖 `update()` 方法名，这是框架保留的。

### 坑2：Sprite方向上下反了

角色按↓走路，屏幕上显示的是背面。查了半天——原来AI生成的sprite sheet行顺序跟代码预设的不一样。

Row 0 是正面（朝下），代码以为 Row 0 是背面（朝上）。一行代码修复：
```js
// 修复前
const dirs = ["up", "right", "down", "left"];
// 修复后
const dirs = ["down", "right", "up", "left"];
```

### 坑3：白底精灵

AI生成的所有sprite图片都有白色背景（或棋盘格背景）。写了个Python脚本，从四个角做flood fill把白色区域变透明。64张图批量处理。

### 坑4：CG图片API超时

GPT Image 2的API在生成大图（1536×1024）时经常524超时。解决：
- 缩短prompt（从150词减到80词）
- 降分辨率（1536→1024）
- 自动重试3次 + 递增等待

### 坑5：升级面板点不了

用Phaser的Container包裹UI元素后，`setScrollFactor(0)` 不能正确传递到子元素的交互检测上。解决：去掉Container，每个UI元素单独设置scrollFactor。

## 代码质量管理

跟AI写代码最重要的事：**你得像技术leader一样管理它。**

我反复说的几句话：
- "模块化，不要屎山"
- "超过200行必须拆"
- "先确认需求再写代码"

效果很好——20个文件，最长200行，每个系统独立，改一个不影响其他。定期重构，代码到现在还很干净。
